Mac下代码审计环境配置

Study
  • Arbitrary File Upload
  • Summary

一直都是在虚拟机下代码审计～感觉还是很麻烦～～倒不如花点时间在物理机上搭个环境一劳永逸好了～

web环境

laradoc + docker

这个相对于下面那个傻瓜式’phpstudy’比较麻烦，但是优点就是可以自定义环境（高级用法）有需要的就去看文档吧。

MxSrvs

这个的确是傻瓜式phpstudy，只需要手动点击开启/关闭/重启，配置文件也可以直接在软件里面修改，唯一的缺点就是，不支持多版本定制，但是基本使用还是没有什么问题的。

PhpStorm + Xdebug

网上在mac端基于MxSrvs用PhpStorm+Xdebug的配置文章～～根本没有～于是只能自己捣鼓，然后记录下来。

配置php.ini

因为MxSrvs每安装一个php版本，里面已经自带xdebug的插件，所以不需要自己下，只需要修改一下配置就可以 ～

打开php.ini，搜索xdebug关键字，在最下面再添加这三行～

xdebug.remote_host : 监听ip
xdebug.remote_port = 9000 : Xdebug监听地址
xdebug.idekey = PHPSTORM： idekey名称（与后面设置对应）

设置php解析器

Preferences -> Languages & Frameworks > PHP

修改为MxSrvs里面php的路径即可

配置 xdebug 参数

设置debug端口

配置代理

配置 xdebug

run -> Edit Configurations

server里面默认是没有的，自行添加对应ip和端口即可。

安装xdebug插件

这里一般默认安装好后，IDE Key是默认配置好的，所以基本上配置就完成了～

测试

发现直接从phpstorm打开是63342端口，然后再去搜一波，发现可以这样改端口到80～

大功告成！！
